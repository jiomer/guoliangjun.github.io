今天在在本地连接linux虚拟机测试代码时，遇到了java.security.NoSuchAlgorithmException: Algorithm DH not available的错误。

原因： DH算法无效。 网上的解决办法为：

  1. 修改java.security文件(我自己觉得不推荐，因为机器不是你的呢?) JDK路径\jre\lib\security\java.security

security.provider.11=org.bouncycastle.jce.provider.BouncyCastleProvider

前面的数字依次增加

2.使用bcprov-jdk16包,将bcprov-jdk16-145.jar 放入 本机 jre/lib/ext 下 即可(试过无效)

我的解决办法是从jdk7切换到jdk8,可能是版本算法不一样(且我项目是用jdk7，但是系统环境jdk8？) 具体可以到下面链接看下咯

ps:
location 是设置查找的目录
–mtime +30 是设置时间为30天前

-type d 这周查找的类型为文件夹
N分钟前 -mini +N

度娘解决的办法是运行一下这个命令：#sed -i 's/\r$//' test.sh

或者vi 脚本，底行模式，输入 set fileformat=unix
